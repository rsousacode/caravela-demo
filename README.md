# caravela_demo

A minimal demo app for [Caravela](../caravela) using the `Library` domain
from Caravela's README (authors, books, publishers).

## Files created by hand

These were written directly to scaffold the app:

| Path | Purpose |
|------|---------|
| [mix.exs](mix.exs) | Project definition; depends on `caravela` via `path: "../caravela"`, plus `ecto_sql`, `postgrex`, `jason`. Adds `ecto.setup` / `ecto.reset` aliases. |
| [.formatter.exs](.formatter.exs) | Imports formatter rules from `:ecto`, `:ecto_sql`, and `:caravela` (so `entity` / `field` / `relation` stay unparenthesised). |
| [.gitignore](.gitignore) | Standard Elixir ignores (`_build/`, `deps/`, crash dumps, etc.). |
| [config/config.exs](config/config.exs) | Registers `CaravelaDemo.Repo`; sets migration primary/foreign keys to `:binary_id` to match Caravela's generated schemas. |
| [config/dev.exs](config/dev.exs) | Local Postgres connection (user `postgres`, db `caravela_demo_dev`, port `5432`). |
| [config/test.exs](config/test.exs) | Test Repo config with `Ecto.Adapters.SQL.Sandbox`. |
| [config/runtime.exs](config/runtime.exs) | Reads `DATABASE_URL` in `:prod`. |
| [lib/caravela_demo.ex](lib/caravela_demo.ex) | Top-level module with a short moduledoc. |
| [lib/caravela_demo/application.ex](lib/caravela_demo/application.ex) | OTP application; supervises the Repo. |
| [lib/caravela_demo/repo.ex](lib/caravela_demo/repo.ex) | `Ecto.Repo` for Postgres. |
| [lib/caravela_demo/domains/library.ex](lib/caravela_demo/domains/library.ex) | The Caravela domain declaration — entities `authors`, `books`, `publishers` and their relations. |
| [docker-compose.yaml](docker-compose.yaml) | Postgres 16-alpine service on `localhost:5432`, user/pass `postgres`/`postgres`, with a named volume and healthcheck. |

## Files generated by Caravela

Produced by `mix caravela.gen.schema CaravelaDemo.Domains.Library` — do
not edit by hand, regenerate instead:

- [lib/caravela_demo/library/author.ex](lib/caravela_demo/library/author.ex)
- [lib/caravela_demo/library/book.ex](lib/caravela_demo/library/book.ex)
- [lib/caravela_demo/library/publisher.ex](lib/caravela_demo/library/publisher.ex)
- `priv/repo/migrations/<timestamp>_create_library_tables.exs`

## Commands run to bootstrap

```bash
# 1. Fetch deps (caravela from ../caravela, plus ecto_sql/postgrex/jason)
mix deps.get

# 2. Compile the bare app to make sure the scaffold is valid
mix compile

# 3. Preview what Caravela would generate from the domain
mix caravela.gen.schema CaravelaDemo.Domains.Library --dry-run

# 4. Generate Ecto schemas and the migration for real
mix caravela.gen.schema CaravelaDemo.Domains.Library --force

# 5. Recompile with the generated schemas in place
mix compile
```

## Running the demo end-to-end

```bash
docker compose up -d        # start Postgres
mix ecto.setup              # create DB + run the generated migration
iex -S mix                  # poke around: CaravelaDemo.Repo.all(CaravelaDemo.Library.Book)
```

To rewrite schemas after editing [lib/caravela_demo/domains/library.ex](lib/caravela_demo/domains/library.ex):

```bash
mix caravela.gen.schema CaravelaDemo.Domains.Library --force
mix ecto.reset
```
